## 2025-03-08 - Added DOMPurify for Defense-in-Depth HTML Sanitization
**Vulnerability:** The application was converting markdown to HTML through string replacements without comprehensive HTML sanitization. While basic HTML tags were escaped through `escapeHtml`, the markdown-to-html transformation introduced complex node structures (including `button`, `pre`, `details`, etc) derived from user-supplied chat data. An attacker could potentially introduce maliciously formatted markdown that would bypass the basic escape rules and render dangerous HTML structures or executable script contexts on the frontend.
**Learning:** Simple HTML escaping of strings (replacing `<`, `>`, etc) prior to markdown conversion is insufficient for preventing XSS when that markdown conversion itself generates complex DOM structures. A dedicated HTML sanitizer like `DOMPurify` is required to ensure the *resulting* HTML string contains only safe tags and attributes before it is injected via `dangerouslySetInnerHTML`.
**Prevention:** In applications that transform user-provided markdown into HTML, the output must be run through a proven, comprehensive HTML sanitizer before being rendered in the browser. Escaping the input string is a good first step, but sanitizing the *output* HTML is critical for defense-in-depth against complex XSS vectors. Always configure the sanitizer with a strict allowlist of elements and attributes.
